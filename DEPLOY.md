## Step 1 ‚Äî Supabase database

1. Go to [supabase.com](https://supabase.com) ‚Üí **New project**
2. Open **Project Settings ‚Üí Database**
3. Copy:
   - **Connection string** (URI mode, port 5432)
   - Convert to JDBC format:
     ```
     jdbc:postgresql://db.YOUR_REF.supabase.co:5432/postgres?sslmode=require
     ```
   - **User:** `postgres`
   - **Password:** your database password

Flyway runs automatically on first API start and creates tables + seed data.

---

## Step 2 ‚Äî Deploy backend (Render)

1. Push this repo to GitHub (already done)
2. Go to [dashboard.render.com](https://dashboard.render.com) ‚Üí **New +** ‚Üí **Blueprint**
3. Connect repo: `isherve/Hardware-Operations-Inventory-Management-System`
4. Render reads `render.yaml` and creates two services
5. Set these **secret** env vars on **bettina-hardware-api**:

| Variable | Example |
|----------|---------|
| `SUPABASE_DB_URL` | `jdbc:postgresql://db.xxx.supabase.co:5432/postgres?sslmode=require` |
| `SUPABASE_DB_USER` | `postgres` |
| `SUPABASE_DB_PASSWORD` | your Supabase DB password |
| `CORS_ALLOWED_ORIGINS` | `https://frontend-mu-wine-23.vercel.app` |

`JWT_SECRET` is auto-generated by Render.

6. Wait for deploy ‚Üí note API URL, e.g. `https://bettina-hardware-api.onrender.com`

Test: `https://bettina-hardware-api.onrender.com/api/health` ‚Üí `{"status":"UP"}`

> **Free tier:** Render spins down after inactivity; first request may take ~30s.

---

## Step 3 ‚Äî Deploy frontend (Vercel)

1. Go to [vercel.com](https://vercel.com) ‚Üí **Add New ‚Üí Project**
2. Import GitHub repo `isherve/Hardware-Operations-Inventory-Management-System`
3. Set **Root Directory** to `frontend`
4. Add environment variable:

| Name | Value |
|------|-------|
| `VITE_API_URL` | `https://bettina-hardware-api.onrender.com/api` |

5. Deploy

6. Copy your Vercel URL and update **Render** `CORS_ALLOWED_ORIGINS` on the API service to match (no trailing slash).

Redeploy API if CORS was wrong on first try.
